# ModelScope-J

A reimplementation of the ModelScope data analysis application in the
[J programming language](https://www.jsoftware.com), served via JHS (J HTTP Server).

ModelScope lets you upload CSV datasets, run Principal Component Analysis,
explore results interactively, and apply k-means clustering — all from a
web browser. This version replaces the original Node.js/React/PostgreSQL
stack with pure J: the same language handles HTTP, computation, and persistence.

---

## What It Does

- **User accounts** — register and log in; credentials stored in SQLite with
  SHA-1 + random salt hashing
- **Dataset upload** — upload a CSV file; the server detects column types,
  computes a quality report (missing values, constant columns, type mix), and
  stores the data
- **PCA** — run Principal Component Analysis via the `jlearn` library's
  `PCASolver` (LAPACK-backed); choose the number of components and preprocessing
  options; results persist in the database
- **Visualisation** — scree plot, 2D/3D scatter of transformed points, loadings
  chart (rendered in the browser via Plotly from CDN)
- **K-means clustering** — apply clustering to any PCA run via `jLearnKMeans`
- **Run comparison** — compare two PCA runs on the same dataset side by side
- **Analysis** — per-column statistics and Pearson correlation matrix

## Getting Started

### Prerequisites

- J9.8 installed at `/Applications/j9.8/`
- The `jlearn` library cloned at `~/jdev/jlearn/`
- Required addons present in `/Applications/j9.8/addons/`:
  - `data/ddsqlite` — SQLite persistence
  - `math/lapack2` — LAPACK for PCA eigendecomposition
  - `tables/csv` — CSV read/write
  - `convert/pjson` and `convert/json` — JSON encoding/decoding
  - `stats/base` — statistical utilities

All of the above ship with J9.8; no separate install step is needed.

### Start the server

```sh
cd ~/jdev/dataview
jconsole -js "load '~/jdev/dataview/jhs.cfg'" > /tmp/modelscope-jhs.log 2>&1 &
```

Then open your browser:

```
http://localhost:65002/mslogin
```

Watch the log:

```sh
tail -f /tmp/modelscope-jhs.log
```

Stop the server:

```sh
kill $(lsof -ti :65002)
```

## Architecture

### Why J and JHS?

The reference implementation (Node.js + React + PostgreSQL) is three languages,
three runtimes, and a build pipeline. This version is one language end to end.
The same J session that parses your CSV, runs PCA via LAPACK, and queries SQLite
also renders the HTML and speaks HTTP. There is no API boundary between the
computation layer and the server layer — they are the same process.

### JHS page model

JHS is a **servlet-style server**, not a static file server. There are no HTML
files. Every page is generated by a J verb in a page locale. The URL path segment
is the locale name: visiting `/mslogin` calls the `jev` verb in the `mslogin`
locale.

Each page locale defines:

- `HBS` — the body template: a string of J sentences, one per line, each
  returning an HTML fragment; evaluated by `jhbs` at render time
- `CSS` — stylesheet rules, inlined into `<style>` tags
- `JS` — client JavaScript, inlined into `<script>` tags
- `jev` — the event dispatcher: handles both initial GET loads (renders the page
  via `jhr`) and AJAX POST events from button clicks (responds via `jhrajax` or
  `jhrcmds`)

```
Browser GET /mslogin
  → JHS evaluates 'jev_mslogin_ 0'
  → jev sees jmid='' (GET), calls 'ModelScope' jhr ''
  → jhr assembles HBS+CSS+JS into full HTML response

Browser clicks 'Sign in' button
  → JS calls jdoajax(['uname','pass'], '')
  → JHS evaluates 'jev_mslogin_ 0' again
  → jev sees jmid='login', jtype='click'
  → calls ms_login, which validates credentials and calls jhrajax
  → JS receives response, navigates to /msdashboard
```

See `docs/JHS-tutorial.md` for a full explanation of the framework.

### Startup

`jhs.cfg` is the entry point. Its `config_jhs_` verb sets the port, opens the
database, and loads the page locale files. The two fixed lines at the end load
`core.ijs` and call `init_jhs_''`, which binds the socket and hands control to
`jfe` (the JHS event loop).

```
jconsole loads jhs.cfg
  → config_jhs_ runs:
      PORT=65002, AUTO=0
      db_open_jhs_ ''       (opens/creates data/modelscope.db)
      load src/db.ijs
      load src/auth.ijs
      load src/mslogin.ijs
      load src/msdashboard.ijs
  → load core.ijs           (JHS framework into jhs locale)
  → init_jhs_ ''            (bind socket, start jfe event loop)
```

### Session state

JHS has no built-in multi-user session mechanism. We maintain two globals in the
`jhs` locale:

| Name | Type | Meaning |
|------|------|---------|
| `MS_UID` | integer | ID of the logged-in user; `_1` when nobody is logged in |
| `MS_USER` | string | Username of the logged-in user; `''` when nobody is logged in |

These are set on login and cleared on logout. Because this is a single-server,
single-process application designed for one local user, this is sufficient.
Two simultaneous browser sessions would share the same globals.

### Database

SQLite via the `data/ddsqlite` addon. Three tables:

| Table | Purpose |
|-------|---------|
| `users` | Accounts: username, password hash, salt, session token |
| `datasets` | Uploaded CSVs: metadata, column info, raw data, quality report |
| `pca_runs` | PCA results: components, explained variance, loadings, transformed points |

The schema is created automatically on first run by `db_ensure_schema`.
The database file is `data/modelscope.db` (gitignored).

## Key J/JHS Notes

A few issues that affect anyone working on this codebase:

**`select./case.` hangs with strings in J9.8.** Use `if./elseif.` chains for all
string dispatch — in route handlers, content-type detection, everywhere.

**`jev` must be defined in each page locale.** JHS calls `jev_mypage_ 0` for every
request to `/mypage`. Without a local `jev` definition, the search path finds the
`jhs` locale's built-in `jev` dispatcher, which cannot see your page verbs.
Always write `jev_get =: jev =: 3 : 0`.

**Every code path through `jev` must call exactly one response verb** (`jhr`,
`jhrajax`, or `jhrcmds`). These all call `htmlresponse` which closes the socket.
Zero calls leaves the browser hanging; two calls errors.

**`ddfet` result shape.** `ddfet sh,_1` returns a boxed rank-2 array. Assign
`nrows =. # rows` to a variable before testing — lazy evaluation of `# rows`
inside `if.` can hang.

See `docs/JHSinfo.md` for the full gotcha reference.
